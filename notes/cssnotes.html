Anatomía de una Regla CSS

Una regla en css está compuesta de 3 partes:

  1. Selector
  2. Brackets
  3. Rules

    Estas "rules", o reglas, se pueden pensar como combinaciones Key: values. Aunque a estas llaves, en el contexto de CSS, se les conoce como Propiedades y cada una tiene una lista de valores que puede tomar. Una propiedad, en forma general puede tomar más de un valor, siendo la sintaxis general: CSSProperty: Value1 Value2 ... ValueN;

Por ejemplo: .a{ Color: Solid Black;}. ".a" es el selector; "Color" es la propiedad y "Solid Black" son los valores asignados a esta propiedad.

Selectores

Un selector es la forma que tiene CSS de identificar dentro del código de HTML a cuales etiquetas les aplicara un estilo

Hay varias clases de selectores.
  • Por etiqueta. Ej: div{contenido}
  • Por clase. Ej: .NombreDeTuClase{contenido}
  • Por Id. EJ: #NombreDeTuId{Contendio}

Combinando selectores

Se puede combinar selectores de forma interesantes
  • selector, selector, ..., selector{} Aplica el estilo a todos los selectores de la lista
  • selector1 > selector2 > ... > selectorn{} Aplica el estilo a los selectores tq: Selector N sea hijo directo de Selector n-1
  • selector1 selector 2 ... selectorn{} Es una relación de contenencia, se parece a la anterior pero Selector n no tiene que ser hijo directo de Selector N-1, solo estar contenido en él.
  • .Class1.Class2. ... .ClassN{} Se aplica a los elementos que tengan todas las clases anteriores definidas.
    Nota: En HTML5, class="abc def" define DOS clases para el elemento, una "abc" y otra "def".

Pseudo Clases

  • Los selectores en CSS pueden tener atributos que simbolizan estados, como, por ejemplo, 'Hover' o 'Active', que solo se activan, respectivamente, si el mouse está sobre el elemento, o si eres un botón y le diste click al elemento pero no has soltado el click. La sintaxis es la siguiente:

Conflict Resolution

This definitivamente starts to get nasty
  • Conflict Resolution by origin
    La idea es que, primero, las StyleSheets externas se piensan como si estuviesen declaradas dentro de un <Style>, en caso de conflictos entre reglas de distintos Style tags, se escoge la que fue delcarada en el Tag declarado de último
  • Merge: Si varias reglas apuntan al mismo target, pero son reglas distintas, simplemente se aplican todas, ya que estas no entran en conflicto entre sí
  • Inheritance: Las reglas aplicadas a un elemento heredable también se le aplican a todos su descendientes, incluso si no son hijos directos. Box model

Se tratará el margin con gris, el border con negro, el padding con azul , y el content con rojo .
Contenido
Básicamente todo elemento de HTML es tratado como una caja, con 4 partes: Margin, border, padding y Content. Cada una es self-explanatory, pero hay que tenerlas en cuenta a la hora de pensar en nuestro diseño y hay que recordar que muchas veces los browsers ponen valores por defecto a cada uno de estos elementos, por lo que, incluso si no hemos definido ningún "grosor", aún así pueden verse que ocupan espacio. Por lo que suele ser buena práctica sobre escibir estos valores si no queremos que el Browwser eliga por nosotros. Para sobreescribir los valores de todos los elementos se deben poner en el selector "*{}", ya que si se ponen en body, solo se sobreescribiran los valores que son heredables.

Box sizing.

Por defecto cuando colocamos la altura y ancho de un elemento, lo que se va a restingir es la altura y el ancho del contenido de las cajas. Por lo que si tenemos Padding, Borders o Margins distintos de 0, las dimensiones totales de nuestra caja será mucho más grandes que las que especificamos. Una way around this, es definir una propiedad llamada "box-sizing: border-box". Esta no es heredable por lo que si la queremos para todo el documento tenemos que ponerla en el selector " *{}"

Responsive Design

Media Queries

Son una forma de agrupar reglas de CSS para que solo sean válidas dependiendo de tamaños específicos en las pantallas del usuario.

La sintaxis es: @media(min/max-width: Npx){ Reglas y selectores usuales}

Se pueden combinar lógicamente con 'and' y ',' para el 'y' y el 'o' lógicos. ej:

@media(min-width: 80px) and (max-width: 800px){ reglas}

BootSrap

BootSrap es un frameWork de CSS desarrollado por Twitter, básicamente divide tu contenido en 12 columnas con 8,33% del tamaño, de la pantalla cada una. Y usando estas columnas como unidad de medida básica, puedes definir macro-columnas que ocupen más de una columna básica. Se definen con clases, Ej: "col-lg-6" ocupatá la mitad de la pantalla

Bootstrap usa una división estándar de Media queris, donde define cinco tipos: extra samll, small, medium, large y extra large:

  • Extra small: < 576px. Ej: "col-6"

  • Small: ≥ 576px. Ej: "col-sm-6"

  • Medium: ≥ 768px. Ej: "col-md-6"

  • Large: ≥ 992. Ej: "col-lg-6"

  • Extra large: ≥ 1200px. Ej: "col-xl-6"
